## Objective

Add a TOON (Token-Oriented Object Notation) export format to `mapx export`. TOON shares JSON's data model but is substantially more compact for LLM consumption: it uses tabular arrays for uniform object lists, inline arrays for primitive sequences, and key folding for deeply nested single-key chains. After I14, `mapx export --format=toon` emits a valid `.toon` document that LLMs can ingest more token-efficiently than the existing Markdown or JSON exports.

This iteration is self-contained and does not block or depend on any other iteration in progress. It can be developed in parallel with I12ŌĆōI15.

## Implementation notes

### TOON quoting edge cases (see F27 for full spec)

The `toonQuote()` helper must handle:
- Values containing the active delimiter (default `,`)
- Values with leading `-` followed by a non-space character (ambiguous with TOON's block scalar marker)
- Values with `null` literal or empty strings
- Control characters in the range `\u0000ŌĆō\u001F` and `\u007FŌĆō\u009F` (escape as `\uXXXX`)

The lone `-` value is a valid TOON null marker and must be output unquoted. Values that start with `-` followed by any non-space character (e.g. `-0.5`, `-active`) must be quoted to prevent ambiguity.

### Token budget trimming

Use the same `pagerank`-based trimming as `LlmExporter`: drop nodes below the `--tokens` budget cutoff, starting from the lowest-importance files. Emit a trailing comment `# N nodes omitted (budget)` when truncation occurs.

### Exporter registration

Register `toon` in `src/exporters/index.ts` alongside `llm`, `dot`, `svg`, `json`.
