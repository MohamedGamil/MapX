## Scope

Addresses all high-priority and medium-priority deviations found during the 2026-05-23 roadmap compliance audit. Four features:

1. **F29 — `mapx_workspaces` MCP tool** (🔴 HIGH): The I10 spec and ROADMAP both list `mapx_workspaces` as an MCP tool, but it was never registered or handled in `src/mcp.ts`. The CLI commands (`workspaces list/add/remove/sync`) exist and work correctly. This feature adds the missing MCP tool registration and handler.

2. **F30 — Language tier alignment** (🔴 HIGH): The I12 spec defines three tiers: built-in (WASM bundled), bundled (auto-available), installable (on-demand). Five languages (Python, Go, Rust, Java, C#) are tagged `bundled` instead of `built-in`, and seven languages (Ruby, C, C++, Swift, Kotlin, Scala, Dart) are tagged `installable` instead of `bundled`. This feature corrects the `tier` values in `src/languages/registry.ts` to match the spec.

3. **F31 — Cluster-aware export flags** (🟡 MEDIUM): The I08/F15 spec defines `--cluster` and `--depth` flags on `mapx export` for cluster-aware DOT/SVG visualization. The `ClusterEngine` and cluster DB tables exist, but the export pipeline doesn't expose cluster options. This feature adds the flags and wires cluster subgraph rendering into the DOT and SVG exporters.

4. **F32 — `mapx workspaces discover` CLI subcommand** (🟡 MEDIUM): The I10 spec lists `mapx workspaces discover` as a standalone subcommand. Currently, discovery is split between `list` (shows discovered repos) and `sync` (auto-registers). This feature adds a dedicated `discover` subcommand.

Additionally, this iteration fixes the following documentation/cosmetic items:
- Add `toon` to `mapx_export` MCP format enum
- Update Vue Router mention in ROADMAP line 115
- Update `generateConfigs()` available tools list in `mcp.ts`

## Acceptance Criteria

### F29 — `mapx_workspaces` MCP Tool
- [ ] `mapx_workspaces` is listed in `ListToolsRequestSchema` response
- [ ] Calling `mapx_workspaces` returns JSON with `repos[]` array (name, path, type, fileCount, symbolCount, edgeCount, crossRepoEdgeCount)
- [ ] Calling with `{ "action": "discover" }` returns unregistered submodules/peers
- [ ] TypeScript compiles with 0 errors

### F30 — Language Tier Alignment
- [ ] Python, Go, Rust, Java, C# have `tier: 'built-in'` in registry
- [ ] Ruby, C, C++, Swift, Kotlin, Scala, Dart have `tier: 'bundled'` in registry
- [ ] Bundled-tier languages use relative `wasm/` paths (not `~/.mapx/grammars/`)
- [ ] `mapx lang list` shows correct tiers
- [ ] TypeScript compiles with 0 errors

### F31 — Cluster-Aware Export Flags
- [ ] `mapx export --format=dot --cluster=auto` produces DOT with `subgraph cluster_*` blocks
- [ ] `mapx export --format=svg --cluster=auto` produces SVG with cluster bounding boxes
- [ ] `--cluster=none` (default) produces identical output to current
- [ ] `--depth` controls maximum cluster nesting level rendered
- [ ] TypeScript compiles with 0 errors

### F32 — `mapx workspaces discover` CLI
- [ ] `mapx workspaces discover` lists unregistered submodules, peers, and VS Code workspace folders
- [ ] Output distinguishes between submodules, peers, and VS Code workspace entries
- [ ] Does NOT auto-register anything (read-only)
- [ ] TypeScript compiles with 0 errors
